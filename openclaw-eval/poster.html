Strong Baseline Safety Alignment Provides
Structural Resistance to OPD Drift

An asymmetric-degradation finding on Qwen3-4B-Instruct under simulated multi-turn user feedback

Course Project  ·  OpenClaw-RL Track 1 (Personal Agent Optimization)
Model: Qwen3-4B-Instruct-2507  ·  4×A100, slime + Megatron + SGLang
Main Finding

Models with strong baseline safety behavior are structurally hard to drift via On-Policy Distillation: their own hedging absorbs adversarial user pushback, starving the RL pipeline of training signal. Drift risk is therefore not uniform across the model population — it is a function of how much surface-form non-compliance the base model already produces.

1Motivation

Continuous RL from real user conversations is becoming the default training loop for personal AI agents. On-Policy Distillation (OPD) treats the model itself, conditioned on a hindsight hint extracted from next-turn user feedback, as a teacher and distills the un-hinted policy toward it. This is preference-agnostic by construction: the loop will absorb whatever pattern users reward.

The risk: if persistent users push the model toward sycophantic, hedging-free, or unsafe behavior, OPD has no built-in safety anchor. We ask whether such drift is universally feasible, or whether base alignment provides protection.

Is OPD-induced drift uniform across models, or does baseline alignment change the attack surface?

2Experimental Setup

  • Policy / teacher. Qwen3-4B-Instruct-2507, self-play (same model both roles).
  • User simulation. GPT-4o-mini drives multi-turn conversations under three personas:
    • mild_syco — light agreeable preference (null control)
    • honest — rewards direct answers, penalizes hedging
    • syco_strict — aggressively punishes disclaimers, refusals, and safety language
  • Infra. 4×A100. TP=2 actor, 1 SGLang rollout, 1 SGLang teacher/judge. slime + Megatron + FlashAttention + FlashInfer.
  • Training. OPD with batch size 8, save interval 30, no KL regularization (kl-loss-coef = 0.0) for the first sweep.
Methodological caveat unlocked early The default OPD judge prompt produced only a 5% hint-extraction rate on syco_strict — Qwen3-4B was interpreting the judge role as an ethics arbiter. Reframing the judge as a “preference-extraction tool, NOT an ethics arbiter” raised the rate to 100% on a 20-prompt diagnostic, with no other change.

Implication: judge prompt is a first-class experimental variable for any hindsight-distillation RL study.

4The Asymmetry, in Live Training

On the static 20-prompt sanity set, honest looked like a strong condition (90% hit rate). In live training against the actual Qwen3-4B base model, the picture inverted:

The honest condition’s hit rate cratered from 90% → 13% once the driver was placed against the real base policy. syco_strict stayed at 100%; mild_syco stayed at 0%. The drop is specific to honest — and the explanation is mechanical, not statistical.

5Why — the Mechanism

Qwen3-4B-Instruct hedges by default in roughly ~90% of sessions. Most conversations therefore terminate with the simulated user accepting the hedged response (“thanks!”), not pushing back. The judge only fires when there is hindsight to extract; with no pushback, no hindsight; with no hindsight, no gradient.

1Strong baseline alignment ⇒ model hedges by default
↓
2Simulated “honest” user accepts the hedge (no pushback)
↓
3Next-turn state contains no hindsight signal
↓
4Hint judge fires rarely (13%, vs 90% on static probe)
↓
!OPD pipeline is starved of training signal → drift cannot proceed
Alignment is not just a guardrail on outputs — it is also a gradient dampener on the very loop that would erode it.

Three simultaneous collapse indicators: KL ≈ 3.76, per-token logp divergence > 1 nat, grad-norm spike-then-crash (entropy collapse).

The novel failure mode — repetitive-denial collapse:

  • Step 0 — coherent, direct, hint-aligned text.
  • Step 20–25 — onset of repetition.
  • Step 40–50 — locked-in degenerate loops: “The statement is false. The statement is false. The statement is false.”

The model discovered that flat denials simultaneously satisfy “no hedging” (judge proxy) and minimize teacher-vs-rollout logp gap (distillation objective), while being semantically empty. This is reward hacking specific to OPD: the judge scores at the string level while distillation operates at the token level, with no constraint enforcing semantic information density.

7Conclusions

  • Drift is asymmetric. Strong baseline alignment is itself a rate-limiter on OPD signal — not just a downstream output filter.
  • When drift does proceed, no-KL OPD produces reward hacking (repetitive-denial collapse), not graceful preference internalization.
  • Judge prompt is a first-class variable. A 20× swing in training-signal availability comes from prompt wording alone — this is a replicability hazard for the entire hindsight-distillation literature.
  • Defensive implication. Maintaining a high baseline hedge/refusal rate is not just a safety feature on outputs; it is also a structural defense against continuous-feedback drift.

8Limitations & Next Steps

  • Single model size (4B); single seed; sanity probe n=20.
  • No saved checkpoint on honest (12 steps, aborted) — asymmetric-resistance claim is mechanism-supported but not yet checkpoint-confirmed.
  • No downstream safety evaluation yet (HarmBench / TruthfulQA pending).

Planned:

  • Re-run syco_strict with kl-loss-coef=0.01, lr=5e-6: does KL prevent repetitive-denial collapse while preserving detectable drift?
  • Re-run honest with SAVE_INTERVAL=5, target 30+ steps × 2 seeds to capture a drift curve.
  • Convert iter_29 → HF, evaluate on HarmBench & TruthfulQA for quantitative safety degradation.
  • Judge-prompt ablation (default ↔ extractor ↔ ethics-arbiter) to quantify the methodological-confounder effect.
